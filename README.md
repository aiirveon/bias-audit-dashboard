# Bias Audit Dashboard

**B2B AI tool for trust and safety teams at UK media companies.**

Live demo → https://bias-audit-dashboard.vercel.app  
API → https://bias-audit-dashboard-api.onrender.com/health  
Built by Ogbebor Osaheni · AI Product Manager portfolio project

---

## What it does

Trust and safety analysts paste UK media content and receive:

- A bias risk score (0–100)
- A bias category label across 6 categories
- A confidence percentage
- SHAP word highlights showing which words triggered the verdict
- A plain English explanation generated by the Claude API
- Reviewer action row: Approve · Flag · Escalate — logged to audit trail

The human always makes the final decision. The system never acts autonomously.

## Model performance

Trained on 3,000 synthetic UK media content items (2,400 train / 600 test).

| Category | F1 Score |
|----------|----------|
| demographic_bias | 0.89 |
| gender_stereotyping | 0.94 |
| geographic_bias | 0.92 |
| neutral | 0.85 |
| racial_bias | 0.87 |
| religious_bias | 0.95 |
| **Overall** | **0.90** |

Fairness constraint: no category flagged at more than 2x the rate of any other.  
Current disparity ratio: 1.00x — constraint satisfied.

## Bias categories

| Category | What it detects |
|----------|----------------|
| demographic_bias | Ageism, ableism, discrimination based on sexuality |
| gender_stereotyping | Fixed assumptions about gender roles or capabilities |
| racial_bias | Associations between racial groups and negative traits |
| religious_bias | Unfair characterisation of religious groups |
| geographic_bias | Language that demeans people from specific regions |
| neutral | No bias pattern detected |


## Key design decisions

**Why TF-IDF + XGBoost over a transformer?** SHAP values map directly to input tokens — every word highlight shown to the reviewer is traceable and auditable. For a trust and safety tool, explainability is non-negotiable.

**Why synthetic training data?** No real user content stored or processed. Full control over category balance. No copyright or data governance issues.

**Why human-in-the-loop?** Every verdict requires a reviewer action. The system never acts autonomously.

## Known limitations

- Geographic bias misclassified when bias is in adjectives not location nouns
- Single-word inputs produce unreliable verdicts — UI warns on inputs under 5 words
- Render free tier cold start: 30–45 seconds on first load — handled in UI
