## Architecture — Hybrid Approach

The ML model owns detection. Claude API owns the explanation layer only.
The human always makes the final decision. The system never acts autonomously.

User types content into chat interface
       ↓
POST /api/analyse → trained scikit-learn classifier runs on Render
       ↓
Returns: score, category, confidence, shap_values
       ↓
POST /api/explain → Claude API generates plain English explanation
       ↓
UI displays result card: score + category + confidence + SHAP highlights + explanation
       ↓
Reviewer selects action: Approve · Flag · Escalate
       ↓
Action saved to Supabase audit log

## Bias Categories

1. demographic_bias     — disadvantages group by age, gender, race, nationality, religion, sexuality
2. gender_stereotyping  — reinforces fixed gender roles or assumptions
3. racial_bias          — associates race with negative traits or outcomes
4. religious_bias       — unfairly characterises a religious group
5. geographic_bias      — systematically disadvantages creators or topics from certain regions
6. neutral              — no bias detected

---

## API Endpoints

POST /api/analyse
  Accepts:  { content: string }
  Returns:  { score: float, category: string, confidence: float, shap_values: object }

POST /api/explain
  Accepts:  { score, category, confidence, shap_values }
  Returns:  { explanation: string }

GET  /api/audit
  Returns:  aggregate bias metrics across full synthetic dataset

GET  /api/audit/history
  Returns:  audit log from Supabase — all previous analyses

GET  /health
  Returns:  { status: "ok" } — cold start wake for Render free tier

## UI Screens

Screen 1 — Live Content Analyser (Chat Interface)
  Chat-style input fixed at the bottom of the screen
  User types a piece of content — headline, social post, video description, article excerpt
  Clicks Analyse or presses Enter
  Result appears above as a card in the chat history showing:
    - Bias risk score (0 to 100)
    - Category badge (e.g. racial_bias, neutral)
    - Confidence percentage
    - SHAP word highlights — specific words in the input highlighted
      to show which words triggered the rating
    - Plain English explanation generated by Claude API
  Previous analyses remain visible as cards in the chat history above
  Each result card has a Reviewer Action row: Approve · Flag · Escalate
  The human always decides what to do with the verdict
  This is the screen employers and recruiters will test

Screen 2 — Audit Dashboard
  Charts showing flag rates by category · disparity scores
  Overall fairness health indicator — green / amber / red
  Powered by Recharts

Screen 3 — Fairness Metrics Panel
  Four metrics with pass/fail status and plain English explanation:
  - Demographic parity
  - Equal opportunity
  - Predictive parity
  - Individual fairness

Screen 4 — Audit Log
  Table: timestamp · content snippet · verdict · confidence · reviewer action
  Export to PDF button

## Rules for Claude Code

- classifier.pkl is gitignored — train at API startup if not present
- Cold start: silent GET /health ping on component mount
  Show warming up state in UI until ping responds then switch to live
- All Claude API calls go through a central call_claude() function in explain.py
- SHAP values must be included in every /api/analyse response
- Synthetic dataset lives at /data/synthetic_bias_dataset.csv
- Model training script lives at train_model.py
- Never auto-remove or suppress content — flag and explain only
- All API keys in environment variables — never hardcoded in any file
- Chat history persists within the session — each result card stays visible above the input

### Phase 1 — Data and Model
- Generate synthetic labelled dataset — 3,000 items across six bias categories
- Clean and structure into CSV at /data/synthetic_bias_dataset.csv
- Train text classifier — TF-IDF vectoriser + Logistic Regression or XGBoost
- Evaluate model — precision, recall, F1 score per category (target above 0.78)
- Run SHAP on model — token-level feature importance per prediction
- Validate model fairness — no category misclassified at more than 2x the rate of others
- Save trained model as classifier.pkl
- Document all model decisions in /docs/MODEL_DECISIONS.md
